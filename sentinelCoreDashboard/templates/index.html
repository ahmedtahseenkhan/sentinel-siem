        Phase 7: Data sources & custom data
        
            
          
              
        • IT Hygiene — IT Hygiene dashboards (OS, software, processes, network; newer versions add Browser Extensions and Services) use state/inventory indices. Use Data Sources here to see index patterns; the same indices can be queried for custom views.
          • 
              
        • Custom data — Identify new sources (app logs, threat intel). Use the command wodle in ossec.conf to run scripts; write custom rules in local_rules.xml to turn output into alerts so they reach the indexer. New indices will appear under index patterns once data is indexed; then add visualizations in this dashboard.
          • 
            
        
          
        

          
        
            
        Phase 8: Operations & troubleshooting
        
            
          
              
        • Maintenance — Check version compatibility (manager, indexer, dashboard) before upgrades; keep cluster nodes on the same version. Schedule regular upgrades.
          • 
              
        • Monitoring — Verify services for your Sentinel stack (manager, indexer, web UI). Check logs (e.g. systemd journal for the web UI and manager logs under /var/ossec/logs/ossec.log). Use Stack Status and Data Sources here to test connectivity.
          • 
              
        • Data issues — Inventory (e.g. software) updates on an interval (e.g. 1 hour); changes can be delayed. Restart the agent service to trigger a scan if scan_on_start is enabled. For raw data, query inventory indices directly.
          • 
            
        
          
        

          
        
            
        Phase 9: Branding, reporting & alerting
        
            
          
              
        • Branding — Customize this Sentinel Core UI by editing templates/ and static/css/style.css (and JS) to match your organization.
          • 
              
        • Reporting — For scheduled PDF reports and email, use your preferred reporting stack (for example, OpenSearch Dashboards reporting) alongside this app, which focuses on live dashboards.
          • 
              
        • Alerts from searches — For saved-search alerting (email, Slack, etc.), use your underlying search/dashboard platform or an external monitoring system; this app does not implement saved-search alerting.
